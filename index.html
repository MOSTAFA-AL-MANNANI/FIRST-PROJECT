6. Quelle est la différence entre GET et POST

GET envoie les données via l'URL (visible), alors que POST les envoie dans le corps de la requête (plus sécurisé, pour formulaires).


7. Que savez-vous de la sécurité web

Validation des données, protection contre les injections SQL, mots de passe hashés, contrôle d'accès par middleware, tokens CSRF...


8. Comment fonctionne un CRUD avec Laravel

  • C : create() + formulaire
  • R : index() + show()
  • U : edit() + update()
  • D : destroy()

Le tout en MVC (Models, Views, Controllers)

10. Quelle est la différence entre SQL et NoSQL

  • SQL : données relationnelles, tables, schéma fixe (ex: MySQL )
  • NoSQL : données non structurées (documents, JSON), flexibilité (ex: MongoDB)

12. API – Interface de Programmation d'Application

Une API est une interface qui permet à deux applications ou systèmes de communiquer entre eux.


13. MVC – Modèle Vue Contrôleur

Un design pattern qui sépare la logique d'une application

  • Modèle : Gère les données (ex. base de données)
  • Vue : Interface utilisateur (HTML/Blade)
  • Contrôleur : Gère les actions et les logiques

14. Auth – Authentification

Le processus de vérification de l'identité d'un utilisateur avant d'accéder à des données ou une page sécurisée.


15. SQL – Structured Query Language Français :

Langage utilisé pour interagir avec des bases de données relationnelles (SELECT, INSERT, UPDATE, DELETE...).


16. React – Bibliothèque JavaScript pour le Frontend

React est une bibliothèque JS utilisée pour construire des interfaces dynamiques et rapides.
Elle est basée sur les composants.


17. PHP – Langage Backend (Serveur)

Langage de programmation exécuté côté serveur. Laravel est un framework PHP moderne.


18. JavaScript – Langage Frontend

Langage exécuté dans le navigateur, utilisé pour rendre les pages interactives. Peut aussi être utilisé côté serveur avec Node.js.

23. C’est quoi un middleware dans Laravel ?

C’est une couche entre la requête et le contrôleur. Il permet d’exécuter une vérification (comme l’authentification) avant d’accéder à une route.

24. Quelle est la différence entre include et require en PHP ?

include : continue l'exécution en cas d’erreur

require : arrête le script si le fichier n’est pas trouvé
